Docker CheatSheet
por Matheus Abreu

Comando

docker build -t nome_da_imagem .

docker run -p 4000:80 nome_da_imagem

docker run -d -p 4000:80 nome_da_imagem

docker exec -it id_container bash
docker ps
docker ps -a
docker start id_container
docker stop id_container
docker kill id_container
docker rm id_container
docker rm $(docker ps -a -q)
docker images
docker rmi nome_da_imagem
docker rmi $(docker images -q)
docker tag nome_da_imagem username/repository:tag
docker push username/repository:tag
docker run username/repository:tag
docker system prune -a

docker volume prune
docker network prune

Uso

Cria uma imagem usando a Dockerfile do diretório atual, sem criar um container
Cria e executa um container baseado na imagem fornecida, mapeando a porta 4000 do computador para a porta 80 do container
A mesma coisa do comando acima, porém roda o container em segundo plano (com a flag -d)
Acopla o terminal em um container em execução
Lista todos os containers ativos no momento
Lista todos os containers, inclusive os inativos
Inicia um container parado
Para um container em execução
Força a parada de um container em execução
Exclui um container (o container não pode estar em execução)
Exclui todos os containers
Lista todas as imagens baixadas no seu computador
Exclui uma imagem (nenhum container pode estar usando ela)
Exclui todas as imagens
Adiciona uma tag à uma imagem, para enviar ao Docker Hub
Envia uma imagem ao Docker Hub pela sua tag
Cria um container baseado numa imagem do Docker Hub
Exclui tudo que não está sendo utilizado no momento (Containers, imagens e redes)
Exclui todos os volumes não utilizados
Exclui todas as redes não utilizadas
